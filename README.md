# 微型空中平台仿真系统目标（仅算法3D展示）

## 1. 项目范围

本项目只实现一个轻量化的“算法结果三维可视化”系统，核心是：

- 前端展示无人机/终端/链路等对象的 3D 动态效果。
- 后端接收前端控制指令，调用算法文件，产出时隙数据并推送给前端。

## 2. 应有功能模块

### 2.1 前端模块（3D展示端）
#### 前端应该在设置端上，应该留有三大功能模块以供点击查看，1仿真场景管理 2仿真参数输入 3仿真子系统，也就是3d展示无人机位置的3维地图模块，同时可以留有接口，实现仿真场景调用算法的开始、暂停和继续、停止的功能模块，下面是具体功能的定义：
1. 仿真场景管理模块
- 提供创建场景的选项，并相应弹出弹窗，和完整系统中的代码一样，要求输入场景的各项信息和参数
- 陈列已经创建的场景信息，并且可以供选择

2. 仿真参数输入模块
- 选择算法类型（提供可以选择的算法类型等）。
- 输入算法参数模块（例如节点数量、移动模型参数、通信半径、仿真时长、时隙大小）。
- 提交参数并触发后端运行。

3. 仿真子系统，地图与三维场景渲染，仿真进行场景的控制模块：
- 加载基础地图（2D/3D底图）。
- 渲染 UAV/HAP/终端/地面站等节点模型。
- 渲染拓扑链路、波束覆盖等可视化元素。
- 初始化仿真。
- 开始、暂停、继续、停止。
- 时间轴/时隙跳转（上一帧、下一帧、自动播放速度调节）。


### 2.2 后端模块（仿真与算法调度端）

1. 仿真控制 API
- 提供初始化、启动、暂停、继续、停止、查询状态接口。

2. 算法加载与执行模块（核心）
- 按统一接口加载算法文件（本地 Python 模块或脚本）。
- 调用算法并产出每个时隙的场景数据。
- 支持切换不同算法实现（插件式）。

3. 时隙数据生成与分发模块
- 统一场景数据结构（节点、链路、波束、热力图、统计指标）。
- 通过 HTTP 拉取或 WebSocket 推送给前端。

4. 运行态数据管理（无数据库）
- 运行时数据保存在内存。
- 可选将结果导出为 JSON 文件（用于回放和复现实验）。

## 3. 技术栈（实现微型系统）

### 3.1 前端

- 核心框架：React + TypeScript
- 3D引擎：Three.js
- 地图引擎：高德地图 JS API（可选）
- UI组件：Ant Design（仅用于配置面板和状态栏）
- 通信：Axios（HTTP）+ WebSocket（实时时隙推送）
- 构建工具：Vite

### 3.2 后端

- 语言与框架：保持原先的springboot架构
- 算法执行：本地 Python 算法文件（统一接口动态调用）(此处先不着急填充)
- 实时通信：FastAPI WebSocket
- 数据组织：Pydantic 模型（规范输入输出结构）
- 进程管理：单进程或后台任务（先简化为单机）

### 3.3 数据与存储策略（明确无数据库）

- 不使用 MySQL/PostgreSQL/MongoDB。
- 仿真过程数据保存在内存。
- 若需留档，仅导出 JSON/CSV 到本地文件。

## 4. 非目标（本阶段不做） 

- 不做用户系统、权限系统。
- 不做完整的场景管理 CRUD 平台。
- 不做多租户与分布式部署。
- 不做大规模历史数据检索与数据库优化。
